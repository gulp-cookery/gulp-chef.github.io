An elegant, intuitive way to reuse gulp tasks

Try it now

Intuitive

By one to one configuration to task mapping, your build works exactly as you would imagine.

Elegant

Using gulp best practices, control tasks execution and combine streams with ease. Keeps things simple yet composable and makes complex tasks manageable yet flexible.

Scalable

Starting your gulpfile.js as usual, then incremental enhance or split tasks when your project grow big.

Sharable

Tasks can be shared as normal node modules via npm.

Non-Intrusive

Not forcing or restricting you to gulp-chef's API.

DRY

By abstracting changes in configuration, you don't repeat your tasks with minor differences among projects.

Getting started is easy.

Replace previous version of gulp with gulp-cli

Gulp-chef requires gulp 4.0. If you have a previous version of gulp installed globally, you need to replace it with gulp-cli. 
npm uninstall -g gulp
npm install -g "gulpjs/gulp-cli#4.0"
Don't worry, the gulp-cli is compatible with gulp 3.x, so you can continue using gulp 3.x for existing projects.

Install gulp 4.0 in your project's devDependencies

npm install --save-dev gulpjs/gulp#4.0

Install gulp-chef in your project's devDependencies

npm install --save-dev gulp-chef

Optionally, install required plugins in your project's devDependencies

npm install --save-dev gulp-ccr-browserify gulp-ccr-postcss browserify-shim stringify stylelint postcss-import postcss-cssnext lost cssnano

Create a gulpfile.js at the root of your project

Run gulp

gulp
